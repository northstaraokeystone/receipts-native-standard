## Requirements

**One of:**
- Production receipts-native implementation
- Significant contribution to standard/tooling
- Deep domain expertise in target area

## Time Commitment

- 1 hour/month (virtual meeting)
- Async GitHub participation
- Vote on proposals within 7 days

## Apply

1. Open GitHub issue using [Join Working Group template](.github/ISSUE_TEMPLATE/join-working-group.md)
2. Include: background, why receipts-native matters to you, what you'd contribute
3. Review within 30 days

## Governance

- **Decisions:** Consensus preferred, majority when needed
- **Standard changes:** 2/3 majority
- **Terms:** 2 years, renewable
- **Anti-capture:** Max 2 members per org

## Contribute

1. **Standard:** Open issue with proposal → 14-day discussion → steering vote
2. **Tooling:** Build in your repo → PR to add to ecosystem
3. **Docs:** Submit PR
